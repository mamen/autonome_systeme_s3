Gehen Sie die Schritte "Getting Started" des ROBOTIS e-Manuals 
http://emanual.robotis.com/docs/en/platform/turtlebot3/overview/#overview 
folgendermassen durch:

0. Hardware Setup, nur wenn Sie einen Roboter zusammenbauen müssen: im Manual als letzter Schritt angegeben, ich würde es als ersten Schritt machen.
   die dort angegebenen Anweisungen zum Zusammenbau sind die gleichen wie die im Kit als DinA4 Heft
   angegebenen. Besser geht es mit dem Heft.
   Es ist alles sehr gut beschrieben. Achten Sie nur eben genau auf die angegebenen Positionen auf der 
   Plattform, in denen jedes Teil seinen Platz finden soll. Nehmen Sie meinen bereits montierten Roboter   
   als Modell, falls Zweifel kommen wo ein Teil montiert werden soll.
   Achten Sie vor allem beim Montieren der Räder an die Motorgehäuse, dass die Schrauben richtig fassen,
   bevor Sie sie festdrehen, da hier schnell das Gewinde durch Schrauben, die nicht richtig fassen, 
   beschädigt werden kann. 

1. PC Setup: genau so wie dort angegeben

2. SBC Setup, nur wenn Sie kein Image vom Micro-SD (hat Franz Geiger) benutzen wollen, ansonsten nur letzter Schritt - ssh ...: 
   - Sie brauchen dazu einen HDMI Bildschirm, eine USB-Maus und ein USB-Keyboard, die Sie mit dem
     Raspberry Pi Board verbinden. Dazu am besten den USB-Stecker für den Lidar Laserscanner heruasziehen  
     (ist für die folgenden Operationen nicht notwendig, später beim Schritt "Bringup", der keine
     Tastatur, Maus und Bildschirm braucht, wieder hineinstecken)
   - wählen Sie die Option "Install Linux based on Raspbian" (6.2.1.2 des e-Manuals)
   - beim Brennen des Distro Images auf die SD card hat bei mir "Etcher" nicht funktioniert,
     deshalb habe ich "other methods to burn" gewählt (dd command unter Linux)
   - gehen Sie ansonsten genau so vor wie ort angegeben
   - probieren Sie am Ende mit "ssh pi@192.168.xxx.xxx" (die richtige IP Adresse Ihres Roboters angeben,
     das geforderte Password lautet "turtlebot") aus, ob eine Verbindung mit SecureShell von Ihrem 
     Notebook (Linux) auf den Roboter funktioniert.

3. OpenCR Setup:
   - ich habe nur den Firmware Update des OpenCR1.0 gemacht, wählen Sie dafür die Methoe #1 (shell script)
   - Installation des Arduino IDEs ist nicht unbedingt nötig, macht Sinn nur wenn man tiefer in das
     Control Board OpenCR1.0 (d.h. in die Thematik von Embedded Systems) eindringen will.

Gehen Sie dann die Schritte von Bringup durch. Immer wenn ein Befehl (alles auf Shell-Ebene) auf dem
Turtlebot durchgeführt werden soll, müssen Sie das über eine zuvor geöffnete SSH-Verbindung tun.
Sie können mehrere solcher Verbindungen offen haben. Wenn Sie einen gestarteten Prozess (meist
ein Ros-Prozess) auf dem Turtlebot beenden wollen, machen Sie das mit CTRL+C.

- bei 7.3 fehlt im Shell-Snippet der Befehl "export TB3_MODEL=burger", der vor 
  "export TURTLEBOT3_MODEL=${TB3_MODEL}" ausgeführt werden muss.

Gehen Sie nun die Schritte von "Basic Operation" durch, von denen nur "Topic Monitor" und 8.2.1 Keyboard Teleoperation sinnvoll sind für uns. Diese beiden Schritte sind auch sinnvoll, um zu prüfen, ob alles 
richtig funktioniert.
Von den "Basic Examples" ist das Beispiel "Obstacle Detection" das wichtige für uns. Schauen Sie sich
den Code von "~/catkin_ws/src/turtlebot3/turtlebot3_example/nodes/turtlebot3_obstacle.py genau an.
Das ist der Ausgangspunkt, wenn Sie selber Controller für das Projekt schreiben wollen.
